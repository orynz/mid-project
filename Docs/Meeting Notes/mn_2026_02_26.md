# 📋 회의록

**주제:** 미디어 추천 챗봇 서비스 개발 (유튜브 기반 학습 서비스)
**일시:** 2026년 2월 26일

---

## 1. 서비스 개요 및 목표 기능

### 1차 목표 (핵심 기능)

- 주제 및 질문 입력 받기
- 관련 유튜브 영상 3개 추천
- 추천 영상 내용 요약 제공 (화면 오른쪽 표시)

### 추가 기능 (시간 여유 시 구현)

- **학습 퀴즈** : AI 에이전트(LangGraph 활용)를 통해 자동 생성, 영어 듣기평가 형식도 고려
- **학습 플랜 생성** : 오늘 할 일, 추천 영상 목록 등 맞춤 학습 계획 제시
- **음성 파일 다운로드** : 제공된 모듈 활용
- **요약 내용 다운로드** : 텍스트 또는 PDF 파일 형태로 제공
- **품질 체크 및 자동 재시도** : 응답 품질 저하 시 또는 접속 오류 시 재처리
- **난이도 조절** : 초급/중급 등 난이도 선택에 따라 검색 결과 재조정

---

## 2. 기술 아키텍처

### 백엔드 구성 방식

- **백엔드 / 프론트엔드 분리** 구조로 구현 (2개 파일 분리 방식 채택 논의, 최종 확정 필요)
- 팀 규모(5인) 및 유지보수 측면을 고려하여 결정
- 선택지: ① 백엔드 + 프론트엔드 2개 파일 분리, ② 백엔드를 DB모듈 + 서버 + 프론트엔드 3개 파일 분리

### 주요 기술 스택

- **FastAPI** : 서버 구축
- **LangChain / LangGraph** : 에이전트 및 노드 설계, 기능별 에이전트 구현
- **프롬프트 엔지니어링** : LLM 요청 제어 (temperature=0 등 파라미터 설계)
- **벡터 DB** : 독립 모듈로 분리하여 관리
- **LLM** : 기존 제공된 백엔드/프론트엔드 코드 기반으로 추천 기능 추가

### LangGraph 노드 설계 방향

- 질문 수신 → 에이전트 처리 → 영상 검색 및 추천 → 요약 출력 흐름으로 구성
- 각 기능(추천, 요약, 퀴즈 등)을 별도 노드로 구현
