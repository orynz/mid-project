### 3. 프로젝트 구조 및 역할 분담

1. **대규모 구분**
    - **백엔드(100 % 담당)** – API, DB, 추천 로직, 메모리·출처 관리
    - **프론트엔드** – UI/UX, 이미지·텍스트 레이아웃, 사용자 입력 처리
2. **세부 작업** (예시)
    - **백엔드**
        - DB 설계·구현 (DB‑담당 1명 이상)
        - API 설계·구현 (API‑담당 1명 이상)
        - 추천 알고리즘·프롬프트 설계 (프롬프트‑담당 2명)
    - **프론트엔드**
        - UI 디자인 (UI‑담당 1명)
        - 프롬프트·검색 결과 표시 구현 (프롬프트‑담당 1명)
        - 이미지·텍스트 통합 출력 (UI‑담당 1명)
3. **협업 방식**
    - **Git** – 기본 버전 관리, 브랜치·리베이스·커밋 필수, 백업·히스토리 관리 강조
